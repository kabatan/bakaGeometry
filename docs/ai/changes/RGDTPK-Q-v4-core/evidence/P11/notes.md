# P11 Notes

P11 implements certificate, support, and replay verification for the current candidate-cover core boundary.

Kernel certificates now bind a certificate hash to the route, plan hash, source relation hashes, output relation hashes, exported variables, and a recomputed binding hash. The binding hash includes the route-specific `KernelCertificatePayload`, so changing the certificate proof data changes the certificate binding. All P7-P10 production kernels construct projection-message certificates with `from_execution_plan_with_payload`. The older `from_execution_plan` constructor remains only as a compatibility constructor and creates `BindingOnly`, which the verifier rejects.

`KernelCertificatePayload` has route-specific variants for target-only support, membership, guarded affine, sparse resultant, target action, regular chain, norm trace, specialization interpolation, and Universal wrapping. Synthetic test certificates and binding-only certificates are explicitly rejected by the verifier and do not derive fully enforced run-certificate invariant flags.

`verify_projection_message` checks package hash, block identity, export authorization, relation variable support, source relation authorization, output relation hashes, route/kernel-kind compatibility, certificate binding, and exact route-specific payload evidence. It then replays the proof payload: target support recomputation, membership certificates, guarded affine substitution steps, resultant certificates, target-action quotient reconstruction plus characteristic coverage/Cayley-Hamilton replay, regular-chain DAG and projection reconstruction, norm-trace tower verification, specialization/interpolation elimination, or Universal inner-payload/membership verification.

SparseResultant replay verifies each resultant certificate in a generated closure: every certificate input must already be an authorized source or earlier generated resultant, and message outputs must match generated raw or primitive resultant hashes. NormTrace replay now includes tower source hashes in payload authorization, requires those source hashes to cover the target expression and every minimal polynomial body, and recomputes tower/step hashes before accepting the tower norm relation.

Source relation hashes are compared in `message.source_relation_ids` order, matching the execution plan binding instead of sorting away plan-order evidence. Payload source polynomial hashes must also be authorized by the kernel context's input system or non-self child messages; a projection message cannot authorize its own forged payload source.

Universal wrapper payloads recursively include `inner_payload` source hashes in this authorization check before exact inner replay. This prevents an outer Universal certificate from presenting authorized wrapper sources while smuggling an otherwise exact but unauthorized inner payload.

`verify_global_support` recomputes the exact squarefree product of target-only composed root relations and accepts only when the supplied support polynomial exactly matches the recomputed support.

`CoreRunCertificate` now records input, canonical system, compression, hypergraph, target projection DAG, kernel plan, projection message, support, squarefree support, root isolation, decoded candidate, global support certificate, invariant flags, and invariant-evidence hashes. `replay_run_certificate` recomputes the problem input hash, canonical system hash, compressed-system hash, relation-variable hypergraph hash, projection-message DAG binding, projection message hashes, kernel-plan hashes from message certificates, support proof from messages, support certificate hash, root hash, candidate hash, invariant evidence, and derived invariant flags.

P11-scoped invariant derivation no longer sets final anti-dispatch/QE-CAD flags to bare `true`. `derive_core_invariant_flags` sets only invariants supported by P11 runtime evidence: verifiable exact-Q payload/support replay, no full-coordinate solution/RUR export from message payloads, and bound projection plans/no hidden fallback at the replay boundary. The broader `no_geometry_dispatch`, `no_problem_id_dispatch`, `no_expected_answer_dispatch`, and `no_qe_cad` fields remain `false` until later acceptance/static anti-drift phases can support those claims.

`replay_run_certificate` derives message-to-message dependencies from each certificate's source relation hashes and payload source hashes, while treating canonical input relation hashes and input polynomial hashes as base-authorized sources rather than child dependencies. It includes derived dependencies in `target_projection_dag_hash`, rejects cyclic dependency graphs, and calls `verify_projection_message` on every retained message with only that message's derived child dependencies. Public kernel `replay` implementations for all nine kernels now delegate to the same exact projection-message verifier through `exact_replay_result`, so the public replay API cannot accept a hash-consistent payload that the certificate verifier would reject. The test suite includes a synthetic certificate with all run hashes recomputed to prove replay still rejects a hash-consistent but semantically unverifiable projection certificate.

P11 focused tests cover:

- Projection message deletion.
- Support, squarefree support, root isolation, decoded candidate, input, canonical hash, compression hash, hypergraph hash, projection DAG binding, kernel-plan, global support certificate hash, and invariant-evidence tamper.
- Global support tamper.
- Synthetic, binding-only, and binding-tampered projection-message certificates.
- Hash-consistent forged TargetAction payloads that alter the output characteristic relation without matching quotient evidence; the forged message is rejected by both `verify_projection_message` and public `TargetActionKrylovKernel::replay`.
- Hash-consistent forged RegularChain payloads that alter projection generators without matching recomputed DAG/projection evidence; the forged message is rejected by both `verify_projection_message` and public `RegularChainProjectionKernel::replay`.
- Hash-consistent forged SparseResultant payloads that alter the message output while carrying unrelated valid resultant certificates; the forged message is rejected by both `verify_projection_message` and public `SparseResultantProjectionKernel::replay`.
- Hash-consistent forged NormTrace payloads that tamper tower source authorization or tower hash; the forged message is rejected by both `verify_projection_message` and public `NormTraceProjectionKernel::replay`.
- Hash-consistent forged Universal inner payloads that use an unauthorized source relation while wrapper sources remain authorized; the forged message is rejected by `verify_projection_message`, public `UniversalTargetEliminationKernel::replay`, and run-certificate replay with hashes recomputed.
- Two hash-consistent target-support messages that can individually replay when each is supplied as the other's child, but whose certificate/payload source hashes form a mutual dependency cycle; run-certificate replay rejects the cycle.
- Two target-support messages with duplicate output/source hashes that are already authorized by canonical input; run-certificate replay accepts this non-cyclic case and does not manufacture dependencies from base-authorized sources.

The first P11 `spec_verifier` checkpoint returned `FAIL_FIXABLE` because the initial implementation was still too hash/binding oriented. The second checkpoint also returned `FAIL_FIXABLE` for TargetAction replay, RegularChain replay, and invariant flag derivation. A later checkpoint returned `FAIL_FIXABLE` because public `kernel.replay` still used structural checks even after `verify_projection_message` performed exact payload replay. The quality review after that returned `FAIL_FIXABLE` because Universal `inner_payload` sources were not recursively authorized. The next spec review returned `FAIL_FIXABLE` because run replay authorized payload sources from every non-self projection message instead of a replayed dependency DAG. The following quality review returned `FAIL_FIXABLE` because dependency derivation could manufacture cycles for source hashes already authorized by canonical input. The next quality review returned `FAIL_FIXABLE` because SparseResultant outputs were not bound to resultant certificate relation hashes and NormTrace tower sources/hashes were under-checked. The boundary review then returned `FAIL_FIXABLE` because replay did not recompute compression/hypergraph hashes and because final anti-dispatch/QE-CAD flags were literal true without P11 enforcement evidence. The remediation added variant-specific proof payloads, exact payload replay, replay-level projection message verification, global-support certificate hash binding, projection-message DAG binding, compression/hypergraph replay checks, invariant evidence hashing, P11-scoped derived invariant flags, replayable quotient input for TargetAction, replayable DAG/source/guard data for RegularChain, generated-closure binding for SparseResultant, tower hash/source binding for NormTrace, non-self source authorization, recursive Universal inner-payload source authorization, base-authorized source handling in derived acyclic projection-message dependency binding, forged-payload negative tests, and exact public kernel replay for all nine kernels.

If the remediated boundary review passes, P11 may close only `MECH-010` and advance `MECH-016`. It does not claim P12 exact root/decode completion, P13 exact-image completion, P14 public orchestration, P15 acceptance stress, P16 final closure, performance readiness, final acceptance, or any R-ID as VERIFIED.
