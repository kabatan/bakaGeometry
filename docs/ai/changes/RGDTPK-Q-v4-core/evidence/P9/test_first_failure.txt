First observed failure:
Compilation failed in the first P9 implementation/test pass because both new P9 kernels passed Vec<SparsePolynomialQ> to max_coefficient_height_bits, which expects &[CanonicalRelationQ].

running 4 tests

Failing tests:
- p9_norm_trace_kernel_exports_verified_tower_support
- p9_regular_chain_kernel_projects_triangular_component

Root cause:
Both kernels produced an exported support relation that was the -1 scalar associate of the expected polynomial. The exact verification path recomputed the same representative, and the zero set is identical, but the tests were too strict about sign.

Targeted fixes before final verification:
- Cost trace coefficient-height input was changed to a local 0 placeholder consistent with existing projection kernels that do not have canonical relation inputs at that point.
- The P9 tests now compare support polynomials up to the rational unit ±1.
